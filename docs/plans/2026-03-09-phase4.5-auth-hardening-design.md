# Phase 4.5: Auth Hardening, Test Data & Demo Strategy Design

**Date:** 2026-03-09
**Status:** Approved

## Goal

Wire Clerk authentication through all API routes and portal pages, seed realistic test data for stakeholder accounts, and establish a demo strategy for unauthenticated visitors.

## Key Decisions

- **Seed before harden:** No active userbase, so seed test data first for immediate login testing, then harden routes. Data linkage uses Clerk user IDs so scoping works automatically when hardening goes in.
- **One auth pattern everywhere:** All protected API routes use `auth()` + `createServerClient()` + org scoping. No exceptions.
- **Templates stay public:** Template browsing is unauthenticated (marketing value).
- **Portal pages auth-gated:** Real portal routes (`/[city]/publisher/...`, `/[city]/government/...`, `/[city]/advertise/...`) require login. Unauthenticated visitors redirect to sign-in.
- **Demo routes are curated tours, not portal mirrors:** Expand `/demo/` with cherry-picked impressive screens per audience. Marketing-oriented, start lean.
- **Demo banner:** Subtle top banner on demo pages: "You're viewing a demo â€” Sign in to access your dashboard" with sign-in CTA and dismiss.

## Layer 2: Auth Hardening

### Pattern (applied to all 13 routes)

```typescript
import { auth } from '@clerk/nextjs/server';
import { createServerClient } from '@/lib/db/supabase-server';

export async function GET(request: NextRequest) {
  // 1. Authenticate
  const { userId, sessionClaims } = await auth();
  if (!userId) return NextResponse.json({ error: 'Unauthorized' }, { status: 401 });

  // 2. Get org context
  const orgType = (sessionClaims?.metadata as Record<string, string>)?.orgType;

  // 3. Use authenticated Supabase client
  const supabase = await createServerClient();

  // 4. Scope queries to user's org
  // Campaigns: .eq('user_id', userId)
  // Orders: through campaign ownership or publisher_id
}
```
